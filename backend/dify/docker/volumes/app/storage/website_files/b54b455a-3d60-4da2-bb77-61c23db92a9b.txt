GitBook](https://www.gitbook.com/?utm_source=content&utm_medium=trademark&utm_campaign=RncMhlfeYTrpujwzDIqw)\n\nWelcome to Dify\n===============\n\nDify is an open-source large language model (LLM) application development platform. It combines the concepts of Backend-as-a-Service and LLMOps to enable developers to quickly build production-grade generative AI applications. Even non-technical personnel can participate in the definition and data operations of AI applications.\n\nBy integrating the key technology stacks required for building LLM applications, including support for hundreds of models, an intuitive Prompt orchestration interface, high-quality RAG engines, and a flexible Agent framework, while providing a set of easy-to-use interfaces and APIs, Dify saves developers a lot of time reinventing the wheel, allowing them to focus on innovation and business needs.\n\n### \n\n[](#why-use-dify)\n\nWhy Use Dify?\n\nYou can think of libraries like LangChain as toolboxes with hammers, nails, etc. In comparison, Dify provides a more production-ready, complete solution - think of Dify as a scaffolding system with refined engineering design and software testing.\n\nImportantly, Dify is **open source**, co-created by a professional full-time team and community. You can self-deploy capabilities similar to Assistants API and GPTs based on any model, maintaining full control over your data with flexible security, all on an easy-to-use interface.\n\n> Our community users summarize their evaluation of Dify's products as simple, restrained, and rapid iteration.\n> \n> \\- Lu Yu, Dify.AI CEO\n\nWe hope the above information and this guide can help you understand this product. We believe Dify is made for you.\n\n### \n\n[](#what-can-dify-do)\n\nWhat Can Dify Do?\n\nThe name Dify comes from Define + Modify, referring to defining and continuously improving your AI applications. It's made for you.\n\n*   **Startups** - Quickly turn your AI ideas into reality, accelerating both success and failure. In the real world, dozens of teams have already built MVPs to get funding or win customer orders through Dify.\n    \n*   **Integrate LLMs into existing businesses** - Enhance capabilities of current apps by introducing LLMs. Access Dify\u2019s RESTful APIs to decouple Prompts from business logic. Use Dify\u2019s management interface to track data, costs and usage while continuously improving performance.\n    \n*   **Enterprise LLM infrastructure** - Some banks and internet companies are deploying Dify as an internal LLM gateway, accelerating the adoption of GenAI technologies while enabling centralized governance.\n    \n*   **Explore LLM capabilities** - Even as a tech enthusiast, you can easily practice Prompt engineering and Agent technologies through Dify. Over 60,000 developers have built their first app on Dify even before GPTs came out.\n    \n\n### \n\n[](#next-steps)\n\nNext Steps\n\n*   Read [**Quick Start**](https://docs.dify.ai/application/creating-an-application)\n     for an overview of Dify\u2019s application building workflow.\n    \n*   Learn how to [**self-deploy Dify**](https://docs.dify.ai/getting-started/install-self-hosted)\n     to your servers and [**integrate open source models**](https://docs.dify.ai/advanced/model-configuration)\n    **.**\n    \n*   Understand Dify\u2019s [**specifications and roadmap**](/getting-started/readme/features-and-specifications)\n    **.**\n    \n*   [**Star us on GitHub**](https://github.com/langgenius/dify)\n     and read our **Contributor Guidelines.**\n    \n\n[NextFeatures and Specifications](/getting-started/readme/features-and-specifications)\n\nLast updated 3 months ago\n\nOn this page\n\n*   [Why Use Dify?](#why-use-dify)\n    \n*   [What Can Dify Do?](#what-can-dify-do)\n    \n*   [Next Steps](#next-steps)\n    \n\nWas this helpful?\n\n[Edit on GitHub](https://github.com/langgenius/dify-docs/blob/main/en/README.md)\n\nThis site uses cookies to deliver its service and to analyse traffic. \n\n[](#common-variables)\n\nCommon Variables\n\n#### \n\n[](#console_api_url)\n\nCONSOLE\\_API\\_URL\n\nThe backend URL of the console API, used to concatenate the authorization callback. If empty, it is the same domain. Example: `https://api.console.dify.ai`\n\n#### \n\n[](#console_web_url)\n\nCONSOLE\\_WEB\\_URL\n\nThe front-end URL of the console web, used to concatenate some front-end addresses and for CORS configuration use. If empty, it is the same domain. Example: `https://console.dify.ai`\n\n#### \n\n[](#service_api_url)\n\nSERVICE\\_API\\_URL\n\nService API Url, used to display Service API Base Url to the front-end. If empty, it is the same domain. Example: `https://api.dify.ai`\n\n#### \n\n[](#app_api_url)\n\nAPP\\_API\\_URL\n\nWebApp API backend Url, used to declare the back-end URL for the front-end API. If empty, it is the same domain. Example: `https://app.dify.ai`\n\n#### \n\n[](#app_web_url)\n\nAPP\\_WEB\\_URL\n\nWebApp Url, used to display WebAPP API Base Url to the front-end. If empty, it is the same domain. Example: `https://api.app.dify.ai`\n\n#### \n\n[](#files_url)\n\nFILES\\_URL\n\nFile preview or download URL prefix, used to display the file preview or download URL to the front-end or as a multi-modal model input; In order to prevent others from forging, the image preview URL is signed and has a 5-minute expiration time.\n\n### \n\n[](#server)\n\nServer\n\n#### \n\n[](#mode)\n\nMODE\n\nStartup mode, only available when starting with docker, not effective when starting from source code.\n\n*   api\n    \n    Start API Server.\n    \n*   worker\n    \n    Start asynchronous queue worker.\n    \n\n#### \n\n[](#debug)\n\nDEBUG\n\nDebug mode, default is false. It is recommended to turn on this configuration for local development to prevent some problems caused by monkey patch.\n\n#### \n\n[](#flask_debug)\n\nFLASK\\_DEBUG\n\nFlask debug mode, it can output trace information at the interface when turned on, which is convenient for debugging.\n\n#### \n\n[](#secret_key)\n\nSECRET\\_KEY\n\nA key used to securely sign session cookies and encrypt sensitive information in the database.\n\nThis variable needs to be set when starting for the first time.\n\nYou can use `openssl rand -base64 42` to generate a strong key.\n\n#### \n\n[](#deploy_env)\n\nDEPLOY\\_ENV\n\nDeployment environment.\n\n*   PRODUCTION (default)\n    \n    Production environment.\n    \n*   TESTING\n    \n    Testing environment. There will be a distinct color label on the front-end page, indicating that this environment is a testing environment.\n    \n\n#### \n\n[](#log_level)\n\nLOG\\_LEVEL\n\nLog output level, default is INFO.\n\nIt is recommended to set it to ERROR for production.\n\n#### \n\n[](#migration_enabled)\n\nMIGRATION\\_ENABLED\n\nWhen set to true, the database migration will be automatically executed when the container starts, only available when starting with docker, not effective when starting from source code.\n\nYou need to manually execute `flask db upgrade` in the api directory when starting from source code.\n\n#### \n\n[](#check_update_url)\n\nCHECK\\_UPDATE\\_URL\n\nWhether to enable the version check policy. If set to false, `https://updates.dify.ai` will not be called for version check.\n\nSince the version interface based on CloudFlare Worker cannot be directly accessed in China at present, setting this variable to empty can shield this interface call.\n\n#### \n\n[](#openai_api_base)\n\nOPENAI\\_API\\_BASE\n\nUsed to change the OpenAI base address, default is [https://api.openai.com/v1](https://api.openai.com/v1)\n.\n\nWhen OpenAI cannot be accessed in China, replace it with a domestic mirror address, or when a local model provides OpenAI compatible API, it can be replaced.\n\n#### \n\n[](#container-startup-related-configuration)\n\nContainer Startup Related Configuration\n\nOnly effective when starting with docker image or docker-compose.\n\n*   DIFY\\_BIND\\_ADDRESS\n    \n    API service binding address, default: 0.0.0.0, i.e., all addresses can be accessed.\n    \n*   DIFY\\_PORT\n    \n    API service binding port number, default 5001.\n    \n*   SERVER\\_WORKER\\_AMOUNT\n    \n    The number of API server workers, i.e., the number of gevent workers. Formula: `number of cpu cores x 2 + 1`\n    \n    Reference: [https://docs.gunicorn.org/en/stable/design.html#how-many-workers](https://docs.gunicorn.org/en/stable/design.html#how-many-workers)\n    \n*   SERVER\\_WORKER\\_CLASS\n    \n    Defaults to gevent. If using windows, it can be switched to sync or solo.\n    \n*   GUNICORN\\_TIMEOUT\n    \n    Request handling timeout. The default is 200, it is recommended to set it to 360 to support a longer sse connection time.\n    \n*   CELERY\\_WORKER\\_CLASS\n    \n    Similar to `SERVER_WORKER_CLASS`. Default is gevent. If using windows, it can be switched to sync or solo.\n    \n*   CELERY\\_WORKER\\_AMOUNT\n    \n    The number of Celery workers. The default is 1, and can be set as needed.\n    \n\n#### \n\n[](#database-configuration)\n\nDatabase Configuration\n\nThe database uses PostgreSQL. Please use the public schema.\n\n*   DB\\_USERNAME: username\n    \n*   DB\\_PASSWORD: password\n    \n*   DB\\_HOST: database host\n    \n*   DB\\_PORT: database port number, default is 5432\n    \n*   DB\\_DATABASE: database name\n    \n*   SQLALCHEMY\\_POOL\\_SIZE: The size of the database connection pool. The default is 30 connections, which can be appropriately increased.\n    \n*   SQLALCHEMY\\_POOL\\_RECYCLE: Database connection pool recycling time, the default is 3600 seconds.\n    \n*   SQLALCHEMY\\_ECHO: Whether to print SQL, default is false.\n    \n\n#### \n\n[](#redis-configuration)\n\nRedis Configuration\n\nThis Redis configuration is used for caching and for pub/sub during conversation.\n\n*   REDIS\\_HOST: Redis host\n    \n*   REDIS\\_PORT: Redis port, default is 6379\n    \n*   REDIS\\_DB: Redis Database, default is 0. Please use a different Database from Session Redis and Celery Broker.\n    \n*   REDIS\\_USERNAME: Redis username, default is empty\n    \n*   REDIS\\_PASSWORD: Redis password, default is empty. It is strongly recommended to set a password.\n    \n*   REDIS\\_USE\\_SSL: Whether to use SSL protocol for connection, default is false\n    \n\n#### \n\n[](#celery-configuration)\n\nCelery Configuration\n\n*   CELERY\\_BROKER\\_URL\n    \n    Format as follows:\n    \n    Copy\n    \n        redis://<redis_username>:<redis_password>@<redis_host>:<redis_port>/<redis_database>\n    \n    Example: `redis://:difyai123456@redis:6379/1`\n    \n*   BROKER\\_USE\\_SSL\n    \n    If set to true, use SSL protocol for connection, default is false\n    \n\n#### \n\n[](#cors-configuration)\n\nCORS Configuration\n\nUsed to set the front-end cross-domain access policy.\n\n*   CONSOLE\\_CORS\\_ALLOW\\_ORIGINS\n    \n    Console CORS cross-domain policy, default is `*`, that is, all domains can access.\n    \n*   WEB\\_API\\_CORS\\_ALLOW\\_ORIGINS\n    \n    WebAPP CORS cross-domain policy, default is `*`, that is, all domains can access.\n    \n\n#### \n\n[](#file-storage-configuration)\n\nFile Storage Configuration\n\nUsed to store uploaded data set files, team/tenant encryption keys, and other files.\n\n*   STORAGE\\_TYPE\n    \n    Type of storage facility\n    \n    *   local (default)\n        \n        Local file storage, if this option is selected, the following `STORAGE_LOCAL_PATH` configuration needs to be set.\n        \n    *   s3\n        \n        S3 object storage, if this option is selected, the following S3\\_ prefixed configurations need to be set.\n        \n    *   azure-blob\n        \n        Azure Blob object storage, if this option is selected, the following AZURE\\_BLOB\\_ prefixed configurations need to be set.\n        \n    \n*   STORAGE\\_LOCAL\\_PATH\n    \n    Default is storage, that is, it is stored in the storage directory of the current directory.\n    \n    If you are deploying with docker or docker-compose, be sure to mount the `/app/api/storage` directory in both containers to the same local directory, otherwise, you may encounter file not found errors.\n    \n*   S3\\_ENDPOINT: S3 endpoint address\n    \n*   S3\\_BUCKET\\_NAME: S3 bucket name\n    \n*   S3\\_ACCESS\\_KEY: S3 Access Key\n    \n*   S3\\_SECRET\\_KEY: S3 Secret Key\n    \n*   S3\\_REGION: S3 region information, such as: us-east-1\n    \n*   AZURE\\_BLOB\\_ACCOUNT\\_NAME: your-account-name eg, 'difyai'\n    \n*   AZURE\\_BLOB\\_ACCOUNT\\_KEY: your-account-key eg, 'difyai'\n    \n*   AZURE\\_BLOB\\_CONTAINER\\_NAME: your-container-name eg, 'difyai-container'\n    \n*   AZURE\\_BLOB\\_ACCOUNT\\_URL: 'https://<your\\_account\\_name>.blob.core.windows.net'\n    \n\n#### \n\n[](#vector-database-configuration)\n\nVector Database Configuration\n\n*   VECTOR\\_STORE\n    \n    *   **Available enumeration types include\uff1a**\n        \n        *   `weaviate`\n            \n        *   `qdrant`\n            \n        *   `milvus`\n            \n        *   `zilliz` (share the same configuration as `milvus`)\n            \n        *   `myscale`\n            \n        *   `pinecone` (not yet open)\n            \n        \n    \n*   WEAVIATE\\_ENDPOINT\n    \n    Weaviate endpoint address, such as: `http://weaviate:8080`.\n    \n*   WEAVIATE\\_API\\_KEY\n    \n    The api-key credential used to connect to Weaviate.\n    \n*   WEAVIATE\\_BATCH\\_SIZE\n    \n    The number of index Objects created in batches in Weaviate, default is 100.\n    \n    Refer to this document: [https://weaviate.io/developers/weaviate/manage-data/import#how-to-set-batch-parameters](https://weaviate.io/developers/weaviate/manage-data/import#how-to-set-batch-parameters)\n    \n*   WEAVIATE\\_GRPC\\_ENABLED\n    \n    Whether to use the gRPC method to interact with Weaviate, performance will greatly increase when enabled, may not be usable locally, default is true.\n    \n*   QDRANT\\_URL\n    \n    Qdrant endpoint address, such as: `https://your-qdrant-cluster-url.qdrant.tech/`\n    \n*   QDRANT\\_API\\_KEY\n    \n    The api-key credential used to connect to Qdrant.\n    \n*   PINECONE\\_API\\_KEY\n    \n    The api-key credential used to connect to Pinecone.\n    \n*   PINECONE\\_ENVIRONMENT\n    \n    The environment where Pinecone is located, such as: `us-east4-gcp`\n    \n*   MILVUS\\_HOST\n    \n    Milvus host configuration.\n    \n*   MILVUS\\_PORT\n    \n    Milvus port configuration.\n    \n*   MILVUS\\_USER\n    \n    Milvus user configuration, default is empty.\n    \n*   MILVUS\\_PASSWORD\n    \n    Milvus password configuration, default is empty.\n    \n*   MILVUS\\_SECURE\n    \n    Whether Milvus uses SSL connection, default is false.\n    \n*   MYSCALE\\_HOST\n    \n    MyScale host configuration.\n    \n*   MYSCALE\\_PORT\n    \n    MyScale port configuration.\n    \n*   MYSCALE\\_USER\n    \n    MyScale user configuration, default is `default`.\n    \n*   MYSCALE\\_PASSWORD\n    \n    MyScale password configuration, default is empty.\n    \n*   MYSCALE\\_DATABASE\n    \n    MyScale database configuration,
